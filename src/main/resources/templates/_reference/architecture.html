技術方式と設計ガイドライン

このサンプルアプリケーションのレイヤ構造・使用技術・設計方針について説明します。

レイヤ構造：三層＋ドメインロジック

このサンプルアプリケーションのレイヤ構造は、 CCSR(Continuous Concurrent Stepwise Refinement)開発手法で取り入れている 三層＋ビジネスロックです。

ビジネスルールに基づく計算・判断を表現するオブジェクトはビジネスロジックに集めます。

使っている要素技術

使っている要素技術は以下の通りです。

  • プレゼンテーション層（画面）：Spring MVC (WebDataBinder), Thymeleaf
  • プレゼンテーション層（API）：Spring MVC (Jackson ObjectMapper), Swagger-UI
  • データベース：MyBatis SQL Mapper, H2 Database, Flyway

設計方針

CCSRの基本的な設計方針は次の通りです。

  • ビジネスロジック（ドメインロジック）を入出力の関心事から独立させる
  • ビジネスロジックを表現するオブジェクト（ドメインオブジェクト）を中心に設計・実装を進める

この設計の考え方とやり方は以下を参考にしてください。
