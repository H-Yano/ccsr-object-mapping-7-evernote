アプリケーションの紹介

ハーブ栽培キットの販売会社が、商品を管理するための社内アプリケーションです。
Thymeleaf-Springのドキュメント(英語) に登場するアプリケーションを参考にしました。

アプリケーションの目的

このアプリケーションはオブジェクトと画面、オブジェクトとデータベース、オブジェクトとJSONデータをマッピングするやり方の学習用に作成しました。

技術方式は、CCSR(Continuous Concurrent Stepwise Refinement)開発手法で使っているレイヤ構造と設計方針に基づいています。 CCSR手法を学ぶためのさまざまな参考資料の一つです。

ドメインモデル

ハーブの栽培キットを表現するドメインモデルです。
このモデルのオブジェクトを、画面・データベース・JSONと双方向でマッピングする方法の説明が、 このアプリケーションの目的です。
本来であれば、ドメインモデルは種をまいた日による出荷タイミングの管理、キットの素材や特徴による価格設定などのビジネスルールを表現すべきですが、 このサンプルでは、具体的なビジネスロジックの記述は省略しています。
